# 气体监测报警小车硬件方案与接线

## 1. 从项目书提炼出的最小可行功能

项目书里真正需要保留的技术主线可以压缩成这几项：

- 甲烷/天然气泄漏检测
- CO 一氧化碳检测
- 温湿度补偿
- 超声波避障
- 移动底盘
- 蜂鸣器报警
- 网络化数据上传和可视化
- 遥控或近程控制

我把它收敛成两种最适合做答辩样机的版本：

## 2. 两个版本怎么选

| 版本 | 适用场景 | 通信方式 | 终端 | 优点 | 不足 |
| --- | --- | --- | --- | --- | --- |
| 云端远程版 | 教室、实验室、有路由器或手机热点 | 小车接入 Wi-Fi，把数据发到服务器 | 任意浏览器 | 最符合“互联网+”项目书表达，可做远程看板 | 依赖 Wi-Fi/服务器 |
| 近程控制版 | 展示、答辩、现场演示 | 小车自己开热点 | 附近手机/电脑 | 不依赖外网，调试快，成功率高 | 不能真正跨地域远控 |

结论：

- 如果你现在首先是“做出一个能演示的成品”，优先做 `近程控制版`。
- 如果你要把“互联网+、平台化、可视化”讲完整，再补 `云端远程版`。

## 4. 为什么我推荐这套组合

### 4.1 主控板

`ESP32-S3-DevKitC-1` 自带 Wi-Fi/BLE，GPIO 资源足够，既能控电机，又能开网页控制面板，避免再加一块树莓派，成本和复杂度都更低。

### 4.2 甲烷传感器

项目书里写的是 `TGS2611`。它本身很贴题，但原始器件更偏工程开发，涉及加热、调理电路、标定曲线。为了先把车做出来，我建议原型阶段改成 `ZC13 UART 成品模块`：

- 5V 供电
- 直接 UART 出浓度
- 预热时间明确
- 少走很多模拟前端调试弯路

如果你后面要把“与项目书完全一致”作为答辩重点，可以把甲烷通道升级成：

- `Figaro TGS2611-E00`
- 或 `Figaro NGM2611-E13` 这类出厂标定模块

### 4.3 CO 传感器

`ZE07-CO` 带温度补偿，UART/模拟双输出，对做原型很友好，比裸电化学探头更适合学生项目。

### 4.4 避障

项目书强调狭小黑暗环境，所以不用红外避障。`URM09` 走 I2C，和 ESP32 接线简洁，量程也够。

## 5. 电源架构

建议这样接：

- 2 节 18650 串联输出约 `7.4V`
- `7.4V -> TB6612FNG VM` 作为电机电源
- `7.4V -> 降压模块 -> 5V`
- `5V -> ESP32 5V`
- `5V -> ZC13`
- `5V -> ZE07-CO`
- `3.3V(ESP32) -> SHT31`
- `3.3V(ESP32) -> URM09`
- 所有模块 `GND` 必须共地

注意：

- I2C 设备如果模块上没有电平转换，尽量按 `3.3V` 供电，避免把 SDA/SCL 拉到 5V。
- 如果你买到的某个 UART 气体模块 TX 是 5V TTL，就要在传到 ESP32 RX 之前加分压或电平转换；文档里的 `ZC13/ZE07-CO` 是 3V TTL 输出，更适合直接接 ESP32。

## 6. 具体接线表

### 6.1 ESP32-S3 与 TB6612FNG

| ESP32-S3 引脚 | TB6612FNG 引脚 | 说明 |
| --- | --- | --- |
| GPIO4 | PWMA | 左电机 PWM |
| GPIO5 | AIN1 | 左电机方向 |
| GPIO6 | AIN2 | 左电机方向 |
| GPIO7 | STBY | 驱动使能 |
| GPIO8 | BIN1 | 右电机方向 |
| GPIO9 | BIN2 | 右电机方向 |
| GPIO10 | PWMB | 右电机 PWM |
| 5V | VCC | 驱动逻辑电源 |
| GND | GND | 共地 |
| 电池 7.4V | VM | 电机电源 |

电机接法：

- 左电机接 `A01/A02`
- 右电机接 `B01/B02`

如果小车前进/后退方向反了，不用改代码，直接交换某一路电机的两根线，或者交换对应 `AIN1/AIN2`、`BIN1/BIN2`。

### 6.2 ESP32-S3 与甲烷模块 ZC13

| ESP32-S3 引脚 | ZC13 引脚 | 说明 |
| --- | --- | --- |
| 5V | Pin1 Vin | 电源 |
| GND | Pin2 GND | 地 |
| GPIO17(TX1) | Pin4 RXD | ESP32 发给模块 |
| GPIO18(RX1) | Pin5 TXD | 模块发给 ESP32 |

ZC13 其他引脚：

- `Pin3` 继电器控制输出
- `Pin6` 电磁阀控制输出
- `Pin7` 蜂鸣器控制输出
- `Pin8` 状态脚

原型版里不强制使用这些输出，先用 ESP32 自己的蜂鸣器报警就够了。

### 6.3 ESP32-S3 与 CO 模块 ZE07-CO

| ESP32-S3 引脚 | ZE07-CO 引脚 | 说明 |
| --- | --- | --- |
| 5V | Pin15 Vin | 电源 |
| GND | Pin5/Pin14 GND | 地 |
| GPIO15(TX2) | Pin7 RXD | ESP32 发给模块 |
| GPIO16(RX2) | Pin8 TXD | 模块发给 ESP32 |

说明：

- 代码按 UART 模式读取。
- 模块还提供 `Pin10 DAC` 模拟输出，但这套代码里没用。

### 6.4 ESP32-S3 与 I2C 模块

| ESP32-S3 引脚 | 模块引脚 | 说明 |
| --- | --- | --- |
| GPIO12 | SHT31 SDA | I2C 数据 |
| GPIO13 | SHT31 SCL | I2C 时钟 |
| GPIO12 | URM09 SDA | 与 SHT31 共总线 |
| GPIO13 | URM09 SCL | 与 SHT31 共总线 |
| 3.3V | SHT31 VCC | 供电 |
| 3.3V | URM09 VCC | 供电 |
| GND | SHT31 GND | 共地 |
| GND | URM09 GND | 共地 |

默认地址：

- `SHT31 = 0x44`
- `URM09 = 0x11`

### 6.5 蜂鸣器

| ESP32-S3 引脚 | 模块引脚 | 说明 |
| --- | --- | --- |
| GPIO14 | 有源蜂鸣器 IN | 报警输出 |
| 5V | VCC | 电源 |
| GND | GND | 地 |

## 7. 代码里默认的逻辑

### 7.1 预热

气体模块上电后先预热 `3 分钟`，这段时间：

- 小车不执行移动命令
- 页面会显示 `preheating=true`
- 只做数据采集，不做有效报警判定

### 7.2 默认报警阈值

当前代码里给的是演示型阈值，后面一定要按实测重新标定：

- `CH4 >= 3000 ppm` 触发预警
- `CO >= 35 ppm` 触发预警
- 任何一个传感器故障位异常也会停机并报警

### 7.3 避障

- 前向距离 `< 25cm` 时，前进类动作会被拦截
- 自动巡航模式下，遇障碍会停止/转向避障

### 7.4 调试时最容易踩的坑

- `ZC13` 和 `ZE07-CO` 这套代码默认按“模块出厂主动上传模式”编写；如果你买到的是被改成“查询模式”的模块，就会出现串口有连线但浓度一直不更新的情况。
- 如果浓度一直为 0，先排查：`供电是否稳定 -> 预热是否足够 -> RX/TX 是否交叉 -> GND 是否共地 -> 模块是否被改成查询模式`。
- 如果小车前进时方向不对，优先交换电机两根线，不要第一时间改代码。
- 如果网页能打开但小车不动，先看是否仍处于 `preheating` 状态，或者是否因为前方距离过近被 `blocked`。

## 8. 你答辩时可以怎么讲

一句话版本：

“原方案中的气体检测、移动巡检、超声波避障、网络化可视化和远程控制都保留了。为了先把样机做出来，主控采用 ESP32-S3，甲烷检测原型阶段采用 UART 成品模块替代原始调理电路，后续再升级为 TGS2611/同类标定模块。”

## 9. 真实落地前必须补的工程项
