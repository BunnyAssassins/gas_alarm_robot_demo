# 答辩 PPT 一页式清单

## 1. 页标题

`基于互联网+的气体监测报警小车原型设计`

副标题可选：

`面向狭小恶劣环境的移动式天然气/有毒气体巡检方案`

## 2. 页面布局建议

- 左上：项目定义 + 核心价值

## 3. 可直接粘贴到 PPT 的精简文案

### 3.1 项目定义

本项目设计了一套面向城市燃气管道、狭小通道和不易检修区域的移动式气体监测报警小车，实现对甲烷/天然气、CO 等危险气体的巡检、报警、数据可视化与远程控制。

### 3.2 核心价值

- 将传统定点检测升级为移动巡检
- 将气体监测与互联网平台结合
- 在危险区域替代人工进入，降低风险
- 支持实时报警、数据查看和远程控制

### 3.3 系统架构

`移动底盘 + 气体检测模块 + 环境感知模块 + 无线通信模块 + 可视化控制平台`

### 3.4 原型核心硬件

- 主控：`ESP32-S3-DevKitC-1`
- 甲烷检测：`ZC13 甲烷模块`
- CO 检测：`ZE07-CO`
- 温湿度：`SHT31`
- 避障：`URM09 超声波`
- 驱动：`TB6612FNG`
- 执行机构：`2WD 小车底盘`
- 报警：`5V 有源蜂鸣器`
- 电源：`2x18650 + 降压模块`

### 3.5 两个版本

| 版本 | 通信方式 | 查看/控制方式 | 特点 |
| --- | --- | --- | --- |
| 云端远程版 | Wi-Fi + 服务器 | 浏览器远程控制 | 突出“互联网+”与平台化 |
| 近程控制版 | 小车热点 AP | 手机/电脑直连控制 | 调试快、演示稳定、不依赖外网 |

### 3.6 已实现功能

- 甲烷与 CO 实时检测
- 温湿度监测
- 超声波避障
- 车体前进/后退/转向
- 超阈值蜂鸣报警
- 远程查看与控制
- 自动巡航演示

### 3.7 演示流程

1. 小车上电并完成传感器预热
2. 页面实时显示 CH4、CO、温湿度、前方距离
3. 用户远程发送前进/后退/左转/右转命令
4. 当浓度超阈值或遇障碍时，小车自动停机并报警

### 3.8 项目亮点

- 保留项目书“气体检测 + 移动巡检 + 互联网平台”的主线
- 硬件成本低，适合学生原型验证
- 具备后续升级到多车联网和云端平台的扩展空间

### 3.9 结论表达

本原型证明了移动式气体巡检设备在城市燃气场景中的可行性，为后续防爆化、平台化和多节点部署提供了基础。

## 4. 一页版极简排版稿

如果你只想做一页特别简洁的 PPT，可以直接用下面这段：

```text
标题：基于互联网+的气体监测报警小车原型设计

项目简介：
面向城市燃气管道和狭小恶劣环境，设计一套可移动、可报警、可联网的气体监测小车，实现危险气体实时检测与远程控制。

系统组成：
ESP32-S3 + ZC13甲烷模块 + ZE07-CO + SHT31 + URM09 + TB6612FNG + 2WD底盘 + 蜂鸣器 + Wi-Fi/热点控制

核心功能：
1. 甲烷/CO 检测
2. 温湿度采集
3. 超声波避障
4. 小车运动控制
5. 声光报警
6. 数据可视化
7. 远程/近程控制

双版本实现：
云端远程版：数据上传服务器，浏览器远程查看和控制
近程控制版：手机连接小车热点，在附近直接控制

项目亮点：
移动巡检替代定点检测
气体监测与互联网平台融合
适合学生原型开发与后续平台扩展
```

“我们把项目书中的核心思路落成了一个小车原型。硬件上采用 ESP32-S3 作为主控，集成甲烷、CO、温湿度和超声波避障模块，完成移动巡检与报警。系统分为两个版本，一个通过 Wi-Fi 接入服务器实现远程控制和数据可视化，一个通过本地热点实现近程稳定演示。这样既保留了互联网+平台化思路，也保证了样机能真正跑起来。”
